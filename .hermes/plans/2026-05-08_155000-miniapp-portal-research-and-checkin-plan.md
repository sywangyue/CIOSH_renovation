## 一、调研结论

### 1.1 WebView 能否聚合门户链接？

**结论：可以，但限制较多，个人主体不可用。**

WebView 组件的关键限制：

| 限制项 | 详情 |
|-------|------|
| **主体类型** | **个人类型小程序不支持 WebView**。必须使用企业主体注册 |
| **每页面唯一** | 每个页面只能有一个 `<web-view>`，且自动铺满全屏，覆盖其他组件 |
| **域名白名单（业务域名）** | 加载的 URL 域名必须在 MP 后台配置到「业务域名」白名单，且域名需已 ICP 备案、支持 HTTPS |
| **动态切换** | `src` 支持动态绑定（`<web-view src="{{currentUrl}}">`），可实现切换不同链接 |
| **门户聚合的形态** | 不能做「多格dashboard多站点同屏展示」；只能做「导航列表→点进某一链接→全屏WebView」的跳转模式 |

**结论**：要实现「门户集合」，架构上不是多WebView同屏，而是做成：
- 首页：聚合链接列表（用原生 Mini Program 组件渲染——图标 + 标题 + 跳转按钮）
- 每个链接点进去：一个全屏 WebView 加载对应的 H5 页面

### 1.2 自有链接是否必须做成 H5 跳转？

**结论：是的。WebView 本身只加载外部网页，不自渲染内容。**

关键路径：

```
小程序页面 (原生组件列表)
  └─ 用户点击链接
       └─ 跳转到 WebView 页面
            └─ <web-view src="https://你的域名/page/xxx">
                   └─ 该 H5 页面承载实际内容
```

这意味着：
- 你**必须**有自己的 Web 服务器来托管 H5 页面
- 域名必须是 **HTTPS + ICP 备案**了
- 域名必须在 MP 后台配置为「业务域名」（需上传 TXT 验证文件到域名根目录）
- 如果链接指向**另一个小程序**，通过 `wx.navigateToMiniProgram` 实现，但需要用户点击+确认弹窗，且平台明确打击「小程序盒子」类聚合推广

如果要聚合的链接来自多个不同通道（如：展商秀 H5、观众预登记 H5、EDM 落地页、活动报名页等）：
- 每个目标来源域名都必须配置到业务域名白名单
- 配置上限：**200 个业务域名**

### 1.3 小程序个人认证流程

**核心结论：不存在「个人认证」这个流程。**

| 主体类型 | 认证方式 | 费用 | WebView | 微信支付 | 服务类目范围 |
|---------|---------|------|---------|---------|------------|
| **个人主体** | 微信实名验证（免费） | 0 | ❌ 不支持 | ❌ 不支持 | 极窄（物流、教育、生活服务等，不含展会/商业服务） |
| **企业主体（未认证）** | 银行对公打款验证（免费） | 0 | ✅ | ❌ 不支持 | 较广 |
| **企业主体（已认证）** | 第三方机构审核 | 300元/年 | ✅ | ✅ | 全类目 |

**对 CIOSH 项目的含义**：

本项目主体是 **杜塞尔多夫展览（上海）有限公司**（MDS），属于企业主体。建议：

1. **注册方式**：企业主体注册（营业执照）
2. **认证方式**：选择微信认证（300元/年），而非对公打款验证
3. **原因**：微信认证后可以开通微信支付（展商报名费、观众门票等），且服务类目更完整

## 二、小程序「门户集合」架构设计（初步）

### 2.1 整体架构

```
微信小程序 ──────────────────────────────────────
│
├─ 首页（TabBar）            ← 原生组件渲染
│   ├─ 展会概览（时间、地点、规模）
│   ├─ 快速入口网格（图标列表）
│   │   ├─ 展商查询 → WebView → H5 展商目录
│   │   ├─ 观众预登记 → WebView → H5 预登记
│   │   ├─ 会议活动 → WebView → H5 活动页
│   │   ├─ 展会直播 → WebView → H5 直播间
│   │   ├─ 供需对接 → wx.navigateToMiniProgram → 对接小程序
│   │   └─ 资料下载 → 原生页面（文件列表）
│   └─ 消息/公告（原生通知）
│
├─ 展商中心（TabBar）        ← 原生组件 + WebView 混合
│   ├─ 我的展位 → WebView → H5 展位管理
│   ├─ 展品管理 → WebView → H5 展品上传
│   ├─ 邀请统计 → 原生页面（数据看板）
│   └─ 会刊信息 → WebView → H5 会刊提交
│
├─ 个人中心（TabBar）        ← 原生组件
│   ├─ 我的报名
│   ├─ 我的收藏
│   ├─ 我的消息
│   └─ 设置
│
└─ WebView 承载层（独立页面）
    └─ 加载各 H5 页面（动态切换 src）
```

### 2.2 技术选型建议

| 模块 | 建议方案 | 原因 |
|------|---------|------|
| 原生页面 | 小程序原生框架（WXML + WXSS + JS） | 个人中心、首页入口等无需 H5 的页面用原生，体验更好 |
| 内容承载 | H5 页面（独立部署服务器） | 展商目录、预登记、活动页等动态内容用 H5 承载 |
| H5 服务器 | 建议自建 Node/Java 服务 + Nginx | 需要 ICP 备案域名，需配置业务域名 |
| 小程序跳转 | `wx.navigateToMiniProgram` | 对接第三方小程序（如中方的小程序） |
| 数据通信 | WebView JSSDK（postMessage） | H5 ↔ 小程序双向通信 |
| 认证/登录 | 微信登录 + 手机号授权 | 原生页面用 wx.login，H5 内用 JSSDK 获取身份 |

### 2.3 需要自建的 H5 页面清单

| H5 页面 | 功能 | 优先级 |
|---------|------|--------|
| 展商目录页 | 展商列表、搜索、详情 | P0 |
| 观众预登记页 | 注册表单、实名认证 | P0 |
| 活动/论坛页 | 活动列表、报名、日程 | P1 |
| 展品展示页 | 展品列表、详情 | P1 |
| 直播嵌入页 | 直播流播放（如对接阿里云直播） | P2 |

## 三、小程序注册认证 Checklist（Check-in 文档）

### 阶段 0：前置准备

- [ ] 确认注册主体：**杜塞尔多夫展览（上海）有限公司**
- [ ] 准备营业执照扫描件（加盖公章）
- [ ] 确定小程序管理员：需有实名微信账号（绑定银行卡）
- [ ] 准备企业邮箱（未注册过微信公众平台）
- [ ] 准备 300 元认证费（企业微信认证费用）
- [ ] 准备 ICP 备案域名（用于 H5 页面托管 + 业务域名配置）
- [ ] 确认小程序名称（2-16 个中文字符，不重名）

### 阶段 1：账号注册

- [ ] 访问 mp.weixin.qq.com → 立即注册
- [ ] 选择账号类型：**小程序**
- [ ] 填写企业邮箱 → 激活邮箱
- [ ] 选择主体类型：**企业**
- [ ] 填写主体信息（营业执照信息）
- [ ] 填写管理员信息（姓名、身份证、手机、微信扫码绑定）
- [ ] 提交 → 等待审核（约 1-3 个工作日）

### 阶段 2：微信认证（建议做，300元/年）

- [ ] 登录 mp 后台 → 设置 → 微信认证 → 申请认证
- [ ] 填写认证资料：
  - 企业营业执照（原件彩色扫描件/照片）
  - 运营者信息（姓名、身份证号、手机号）
  - 企业银行账户信息（对公）
  - 联系邮箱
- [ ] 选择认证类型：**微信认证（300元）**
- [ ] 在线支付 300 元认证费
- [ ] 等待第三方审核机构联系（通常 1-5 个工作日）
- [ ] 审核通过 → 获得认证标识

### 阶段 3：开发配置

- [ ] 开发设置 → 服务器域名（配置 wx.request 允许的域名）
- [ ] 开发设置 → **业务域名（配置 WebView 允许的域名）**
  - 上传验证文件到每个域名的根目录
  - 已配置域名上限：最多 200 个
- [ ] 开发设置 → AppSecret 获取并妥善保存
- [ ] 开发设置 → 消息推送配置（对接客服消息）
- [ ] 添加开发者（开发者工具扫码授权）

### 阶段 4：功能开发准备

- [ ] 选择服务类目（展会属于「商业服务-会展服务」或「生活服务」）
- [ ] 注册微信商户号（如需微信支付，认证后操作）
- [ ] 配置用户隐私保护指引（小程序管理后台）
- [ ] 配置用户协议 & 隐私政策页面

### 阶段 5：提审与发布

- [ ] 使用微信开发者工具开发 → 真机调试
- [ ] 上传代码到 MP 后台
- [ ] 提交审核（填写审核备注，说明小程序用途）
- [ ] 审核期间准备：用户手册、客服响应机制
- [ ] 审核通过 → 发布（灰度发布 → 全量发布）
- [ ] 配置小程序码、分享卡片等推广素材

## 五、关键风险与待确认事项

### 已知风险

| 风险 | 等级 | 说明 |
|------|------|------|
| 个人主体不满足需求 | 已关闭 | 确认必须走企业主体，WebView + 支付均不支持个人 |
| H5 服务器需 ICP 备案 | ⚠️ 中 | MDS 现有域名是否已备案？如无，备案需 15-20 个工作日 |
| 业务域名白名单限制 | ⚠️ 中 | 每个域名需上传验证文件，跨部门协调可能耗时 |
| 双方数据权限 | 🔴 高 | 小程序需要的中方数据（展商库、观众库），获取协议需提前敲定 |
| 小程序名称抢注 | ⚠️ 低 | "劳保展"等相关名称可能被占用，需备选 2-3 个 |

### 待确认事项

1. **数据来源**：小程序内展示的展商数据、观众数据，来自哪里？
   - 自建数据库？对接中方数据库？对接慕渊 API？
2. **认证协议**：MDS 是否有现成的 ICP 备案域名可用于 H5 服务？
3. **团队资源**：H5 + 小程序开发由内部还是外包？是否有前端资源？
4. **中方协同**：小程序是否需要中方共同管理？（如展商信息维护由中方负责）
5. **小程序名称**：CIOSH 小程序叫什么名字？（「CIOSH劳保展」？「中国劳保展」？）

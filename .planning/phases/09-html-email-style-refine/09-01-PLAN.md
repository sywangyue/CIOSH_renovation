```python
def _priority_block(
    items: list[dict[str, Any]],
    border_color: str,
    bg_color: str,
    seq_start: int,
) -> tuple[str, int]:
    """
    Wraps items in a left-border colored block for Part B.
    Returns (html_str, next_seq_num).
    If items is empty, returns ("", seq_start) — no wrapper div rendered.
    """
    if not items:
        return ("", seq_start)
    rows = []
    for i, item in enumerate(items):
        seq_num = seq_start + i
        is_last = (i == len(items) - 1)
        rows.append(_item_row_b(item, seq_num, is_last))
    block = (
        f'<div style="border-left:4px solid {border_color};'
        f'background:{bg_color};padding:4px 12px;margin-bottom:12px;">'
        + "".join(rows)
        + f'</div>'
    )
    return (block, seq_start + len(items))
```

Constraints:

  so the wrapper div is entirely omitted (cleaner output per D-01).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Injection (XSS) | `_item_row_b` summary_zh / `_item_row_c` title | accept | `_html.escape()` is already called on all DB-sourced text fields before insertion into HTML. This phase does not add any new unsanitized interpolation points. No change to existing protection. |
| T-09-02 | Injection (XSS) | `_priority_block` → bg_color / border_color params | accept | These params are passed from module-level constants (GREEN, ORANGE, MUTED, GRAY_BG, LIGHT_GREEN, LIGHT_ORANGE) — all hard-coded strings, never from external input. No escape needed. |
| T-09-03 | Information Disclosure | seq_num leaks item count | accept | Email is sent to internal recipients only (MAIL_TO/MAIL_CC env vars). Item count already shown in header. No new information exposure. |
| T-09-04 | Tampering | source_keyword sort key from DB | accept | Worst case: items reordered within a priority group. source_keyword is not displayed in the email — only used for sort order. No integrity risk. |
</threat_model>
